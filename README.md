# CSA_Coursework
## API overview
This RESTful API is designed to serve as the backbone for a Smart Campus ecosystem, providing a robust interface for monitoring and managing physical infrastructure. The system allows for the hierarchical management of campus Rooms, the Sensors assigned to them, and the real-time Readings generated by those sensors.  
The API adheres to industry-standard architectural patterns to ensure it is both "leak-proof" and easy to navigate.

### Core Architectural Principles
- **RESTful Maturity (HATEOAS):** Unlike traditional static APIs, this system implements Hypermedia as the Engine of Application State. Responses include navigational links, allowing client developers to discover available actions dynamically without relying solely on external documentation.  

- **Sub-Resource Mapping:** The API utilises the Sub-Resource Locator pattern. This modular approach delegates logic to specialised classes (e.g., `SensorReadingResource`), ensuring a clean separation of concerns and making the codebase easier to maintain as the campus infrastructure grows.  

- **Global Error Handling:** A centralised Global Safety Net intercepts all internal exceptions. This prevents the exposure of sensitive Java stack traces to external users, instead providing sanitised, semantically accurate JSON error responses (such as "422 Unprocessable Entity" for logic errors).  

### Technology Stack  

|Component|Technology|
|---|---|
|Language|Java 17+|
|Framework|JAX-RS (Jersey Implementation)|
|JSON Provider|Jackson|
|Server|Apache Tomcat / GlassFish|
|Architecture|REST (Representational State Transfer)|

  
### Key Capabilities  

1. **Infrastructure Discovery:** Navigate from the root API to specific rooms and their associated hardware.  

2. **Environmental Monitoring:** POST and GET sensor readings to track metrics like temperature, humidity, and CO2 levels.  

3. **State Management:** Update sensor statuses (e.g., active vs. maintenance) with built-in validation to prevent data entry during downtime.  

4. **Safe Deletion:** Idempotent removal of resources with conflict-checking to ensure rooms aren't deleted while sensors are still active.

## Setup Instructions
**Prerequisites**
	- Java JDK 8 or higher
	- Apache Maven 3.x
	- Apache Tomcat 9.x
	- Apache NetBeans IDE (recommended) or any Maven-compatible IDE

1. Download the project zip file from the GitHub.
2. Unzip the folder.
3. Open NetBeans and go to File > open project and navigate to the directory where you have unzipped the project. Click on the project folder and press "Open Project" button.
4. After the project has been added to the NetBeans, open the left sidebar. Right click on the project's root and click the "Run" button. 

## Curl commands

1) View api metadata
```curl -s http://localhost:8080/CourseWork/api/v1 | jq```

2) View list of rooms
```curl -s http://localhost:8080/CourseWork/api/v1/rooms | jq```

3) Create a new room
```
curl -s -X POST http://localhost:8080/CourseWork/api/v1/rooms \
     -H "Content-Type: application/json" \
     -d '{
        "id": "LAB-100",
        "name": "Laboratory",
        "capacity": 40
     }' | jq
```

4) Get information of a specific room
```
curl -s -X GET http://localhost:8080/CourseWork/api/v1/rooms/LIB-301 \
     -H "Accept: application/json" | jq
```

5) Delete a room
`curl -i -X DELETE http://localhost:8080/CourseWork/api/v1/rooms/LAB-100`

6) Post a new sensor
```
curl -s -X POST http://localhost:8080/CourseWork/api/v1/sensors \
     -H "Content-Type: application/json" \
     -d '{
        "id": "TEMP-300",
        "type": "Temperature",
        "status": "MAINTENANCE",
        "roomId": "LIB-301"
     }' | jq
```

7) Get the list of all the sensors  ```
```
curl -s -X GET http://localhost:8080/CourseWork/api/v1/sensors \
     -H "Accept: application/json" | jq
```

8) Get the list of sensors of a specific type
```
curl -G -s http://localhost:8080/CourseWork/api/v1/sensors \
     -d "type=CO2" \
     -H "Accept: application/json" | jq
```

9) Get sensor readings of a specific sensor
```
curl -s -X GET http://localhost:8080/CourseWork/api/v1/sensors/TEMP-101/readings \
     -H "Accept: application/json" | jq
```

10) Post a new sensor reading
```
curl -s -X POST http://localhost:8080/CourseWork/api/v1/sensors/TEMP-101/readings \
     -H "Content-Type: application/json" \
     -d '{
        "value": 25.5
     }' | jq
```

11) Attempting to delete a Room that still has Sensors assigned to it
```
curl -i -X DELETE http://localhost:8080/CourseWork/api/v1/rooms/LIB-301
```

12) Attempting to POST a new Sensor with a roomld that does not exist
```
curl -i -X POST http://localhost:8080/CourseWork/api/v1/sensors \
     -H "Content-Type: application/json" \
     -d '{
        "id": "TEMP-300",
        "type": "Temperature",
        "status": "ACTIVE",
        "roomId": "LAB-701"
     }'
```

13) Attempting to add a new sensor reading to a sensor under maintenance
```
curl -i -X POST http://localhost:8080/CourseWork/api/v1/sensors/TEMP-300/readings \
     -H "Content-Type: application/json" \
     -d '{
        "value": 26
     }'
```

14) Testing unexpected runtime errors
```
curl -i -X GET http://localhost:8080/CourseWork/api/v1/roomss
```
